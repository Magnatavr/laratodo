# LaraTodo

LaraTodo — простое приложение для управления задачами на Laravel. Позволяет создавать, редактировать, удалять задачи, прикреплять файлы, добавлять теги, управлять приоритетами и статусами.

---

## Основные возможности

* CRUD для задач (создание, просмотр, редактирование, удаление)
* Управление статусом задачи: `В ожидании`, `В процессе`, `Выполнено`, `Архив`
* Приоритеты задач: `Обычный`, `Низкий`, `Высокий`, `Срочный`
* Возможность прикреплять файлы (вложения)
* Привязка тегов к задачам
* Поддержка дедлайнов
* Логирование действий и ошибок
* Пагинация задач
* Уведомления и напоминания (при необходимости)

---

## Требования

* PHP >= 8.1
* Laravel >= 10
* Composer
* MySQL или другая совместимая база данных
* Node.js и npm (для сборки фронтенда, если требуется)

## Установка

1. Клонируем репозиторий:

```bash
git clone https://github.com/yourusername/laratodo.git
cd laratodo
```

2. Устанавливаем зависимости:

```bash
composer install
npm install
npm run dev
```

3. Создаём `.env` на основе `.env.example`:

```bash
cp .env.example .env
```

4. Настраиваем подключение к базе данных в `.env`:

```dotenv
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=laratodo
DB_USERNAME=root
DB_PASSWORD=
```

5. Генерируем ключ приложения:

```bash
php artisan key:generate
```

6. Выполняем миграции и сиды (опционально):

```bash
php artisan migrate --seed
```

7. Создаём символьную ссылку для хранения файлов:

```bash
php artisan storage:link
```

8. Запускаем локальный сервер:

```bash
php artisan serve
```

Теперь приложение доступно по адресу: `http://127.0.0.1:8000`

## Использование

* Создание задачи: `/tasks/create`
* Просмотр задач: `/`
* Редактирование задачи: `/tasks/{id}/edit`
* Удаление задачи: кнопка на странице задачи
* Фильтрация задач по статусу, тегам, поиску
* Пагинация для удобного просмотра большого количества задач

---

## Логирование

Все ошибки логируются в файл:

```
storage/logs/laravel.log
```

Уровень логирования по умолчанию: `debug`.

Для проверки работы логов можно использовать:

```php
use Illuminate\Support\Facades\Log;

Log::info('Тестовое сообщение');
```

## Структура проекта

```
app/
├── Http/Controllers/Task/  # Контроллеры для задач
├── Models/                  # Модели: Todo, Tag, Attachment и др.
├── Services/Todo/           # Сервисный слой для логики работы с задачами
resources/
├── views/
│   ├── tasks/               # Шаблоны для CRUD задач
│   └── layouts/             # Основной layout
routes/
├── web.php                  # Веб-маршруты
```

## Настройка файлов и вложений

* Все прикреплённые файлы хранятся в `storage/app/public/attachments`
* Для доступа из браузера создайте символическую ссылку:

```bash
php artisan storage:link
```
